# User guide

1. Open the simulator and confirm the terminal is READY.
2. Pair a demo POS with the generated one-time code if testing API sessions.
3. Choose a fictional card and scenario, or keep the approved defaults.
4. Enter the amount and optional tip/service values.
5. Follow the presented tap, insert, swipe, or PIN interaction.
6. Review the result, receipt, transaction history, and developer JSON.
7. On Android or Windows, start the local API in Settings or API Monitor.

Scenario Studio can control delay, terminal states, PIN behavior, signature,
processor text, receipt behavior, and a validated custom JSON response. Reset
the simulator to clear temporary sessions, settings, and transactions.
