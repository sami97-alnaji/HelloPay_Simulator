## Defects found and fixed

The physical matrix exposed two real release blockers:

1. Refund, void, void-ID mismatch and Close Batch existed in the shared engine
   but were not reachable as actual Android operations. Transaction detail now
   exposes the operations, result rendering retains operation-specific wording,
   and Close Batch opens a settlement report generated by the shared engine.
2. The PIN-blocked scenario displayed/used selected-card behavior instead of
   the effective scenario override. PIN submission and the screen now use the
   same effective behavior and transition to the blocked result.

Five regression tests were added. The final suite increased from 50 to 55 and
all tests passed.
